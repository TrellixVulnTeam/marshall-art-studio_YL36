Application Properties

The following describes available application properties:

Property Name Description
ACTIVATE_PAS_SIMULATION

Property dictates whether the policy administration simulation needs activated. This is sued when endorsement or renewal is created from a new business work item.

EXAMPLE:

ACTIVATE_PAS_SIMULATION=true
artifact_id_record_lock_time_in_seconds

Number of seconds a resource is locked after a user is no longer making changes to a resource in the toolkit. If a user closes their browser window while a toolkit resource is locked, the resource remains locked from the last update, plus a number of seconds configured using this property. Default is 600 (10 minutes) and blank. 0 or a negative number turns resource locking off; however, this is not recommended.

EXAMPLE:

artifact_id_record_lock_time_in_seconds=600
auto_save_interval

The amount of time in seconds between checks to auto save data. 0 or less turns auto saving off. The value defaults to 60 if no property is provided.

EXAMPLE:

auto_save_interval=60
use_auto_save_on_save

Turns on the auto save process when a user clicks Save or Save and Exit and the value is true. You can only set this property to true if the auto_save_interval property is greater than 0.

The auto save process is turned off when the value is false (this is the default).

EXAMPLE:

use_auto_save_on_save=false
autoMaintainIdAttributes

This entry engages the automatic id attribute maintenance at application scope when the value is true.

EXAMPLE:

autoMaintainIdAttributes=true
boot_service_providers

This lists several built-in service providers that require some activity to take place during application boot up, such as the HTMLTemplateManager and the TransactionDefinitionManager. User supplied classes or subsystems are also listed here.

EXAMPLE:

boot_service_providers=\
com.agencyport.html.elements.HTMLTemplateManager;\
com.agencyport.html.optiongen.OptionProviderManager;\
com.agencyport.domXML.web.APDOMFactoryManager;\
com.agencyport.trandef.TransactionDefinitionManager;\
com.agencyport.connector.ConnectorProcessor;\
com.agencyport.pdf.PDFLicenseManager;\
com.agencyport.widgets.SpecialFieldHelperFactory;\
com.agencyport.arcrule.APadaptors.ArcRuleInitializer
bootup_validate_schemas

This disables validating schemas when booting the application. For use when there is no internet access on the server to reduce start up time. Default is true.

EXAMPLE:

bootup_validate_schemas=
builtin_field_vals_connector

The default SDK field validation connector manager (com.agencyport.fieldvalidation.connector.BuiltinFieldValConnectorManager) is sufficient for a majority of circumstances; however, if preconditions that determine whether to run the connector need modified, then an application can decide to extend the com.agencyport.fieldvalidation.connector.BuiltinFieldValConnectorManager base class and override the continueWithExecute method.

You can configure this derived form at application scope and will apply to all transactions. Only in cases where the default implementation in the SDK needs extended or overridden do you need to provide the following property.

EXAMPLE:

builtin_field_vals_connector=com.carrier.custom.validation.CustomBuiltInValidator
client_debug

The property client_debug value is passed to the client side Java script. A true value enables the AgencyPortal debug console.

EXAMPLE:

client_debug=true
client_update_interval

Controls how often the client sends periodic updates to the back end to check work item locking and work item assistant updates. The default is 15 seconds. The minimum value allowed is also 15 seconds.

EXAMPLE:

client_update_interval=15
client_update_timeout

This property controls the length of time that the client can idle before the period updates that check work item locking and work item assistant updates are stopped. The default is 600 seconds.

EXAMPLE:

client_update_timeout=600
commands

This list tells, for a given page, which servlet category to run. Initially, it displays backwards at first, until its purpose is understood. The URL portion of the servlet (command) to run for a given page is dynamically generated at runtime. The name is based on the name found in the page's id and certain other factors (see the Transaction Definition File reference). You can refer this name as the target servlet.

The framework looks in the override class list for a match to the target servlet name. If it finds a match, it knows which base class (standard framework supplied class) to execute. However, when a page requires special processing, the name of the user-created servlet cannot be found in the override list and the framework attempts to load the actual target class, not a base class.

EXAMPLE:

com.agencyport.servlets.persauto.CMDDisplaydisplayFile=\
com.agencyport.servlets.shared.CMDDisplaydisplayFile\
com.agencyport.servlets.persauto.endorse.CMDDisplaylaunchPASRequest=\
com.agencyport.servlets.base.CMDBaseLaunchPASRequest\
com.agencyport.servlets.workerscomp.endorse.CMDDisplaylaunchPASRequest=\
com.agencyport.servlets.base.CMDBaseLaunchPASRequest
DATA_SCHEMA_RESOURCES

Semi-colon delimited list of schemas available to a class loader, or is a URL. An application can use this technique if they choose to bundle resources in a jar file.

EXAMPLE:

DATA_SCHEMA_RESOURCES=\
http://wiki.keyonce.com/documents/AgencyPortal/schemas\
/transactionDefinitionBehavior.xsd\
[Element:transactionDefinitionBehavior];\
${CHOICEPOINT_SCHEMA_FILE}[Element:NCFRequest,Element:NCFResponse]\
${CHOICEPOINT_SCHEMA_FILE}[Element:NCFRequest,Element:NCFResponse]
DATA_SCHEMAS

Semi-colon delimited list of schemas for an application to load.

EXAMPLE:

DATA_SCHEMAS=\
${my_context_path}WEB-INF/schemas/${ACORD_FILE_NAME_1_7_0}\
[Element:PersAutoPolicyQuoteInqRq,Element:WorkCompPolicyQuoteInqRq];\
${my_context_path}WEB-INF/schemas/${PRECONDITIONS_SCHEMA_FILE_NAME}\
[Element:PersonalAutoPreConditions]
database_agent_class_name

The database_agent_class_name specifies the SDK database agent class to use for the application.

EXAMPLE:

database_agent_class_name=com.agencyport.database.SQLDatabaseAgent
database_password

Password used by the jdbc driver to connect to the database:

EXAMPLE:

database_password=pass123
database_url

URL used by the jdbc to connect to the database.

EXAMPLE:

database_url=jdbc:sqlserver://127.0.0.1:1433;databaseName=\
agencyportal_4_0;selectMethod=direct
database_username

Username used by jdbc driver to connect to the database.

EXAMPLE:

database_username=user123
datasource

The datasource property identifies the JNDI name that refers to the database connection parameters.

EXAMPLE:

datasource=java:comp/env/AgencyPortal_Demo_db_sqlserver

(java:comp/env/ is the data source prefix and AgencyPortal_Demo_db_sqlserver is the JNDI name)
db_table_prefix

You may need to add table prefixing in some installations when the user id that is provided in the JNDI is not owner of the tables or stored procedures. The following table prefix setting instructs the SDK database framework to apply the specific prefix in front of table names. If you use this facility, it is important to add the dot (.) at the end of the prefix value.

EXAMPLE:

db_table_prefix=agencyportal

(you can use the table prefix in the query as "Select * from ${db_table_prefix}tablename")
default_xml_option_list_provider_alias

This property allows you to create a short name for the xml option reader to use in the transaction file instead of the full name.

EXAMPLE:

default_xml_option_list_provider_alias=xmlreader

You can then use the short name in the application, such as the following:

optionList="xmlreader:codeListRef.xml:state"
defer_logging_initialization

This property instructs the AgencyPortal framework to defer all logging configuration to the web container.

EXAMPLE:

defer_logging_initialization=true
DOCUMENT_STORE_SIZE

The XML data structure holding the transaction date (APDataCollection) is cached in memory. Set this property to the approximate number of concurrent users, unless that number is very large. Set this property to 0 (zero) in clustered configurations. The default value when this property is missing is 0.

EXAMPLE:

DOCUMENT_STORE_SIZE=0
fileattachment_cache_directory

Specifies the cache directory used for file attachments. The user should create this directory.

EXAMPLE:

fileattachment_cache_directory=/tmp/fileattachment_cache
html_template_file

The location of an extension html_element_definitions.txt file. An extension is only necessary if an application wants to alter one or more templates defined in the SDK's copy.

The SDK maintains an immutable copy of this configuration file in apwebapp.jar. If an application requires an alteration of one or more template names in the SDK's version, the following configuration is loaded on top of the SDK's version, overlaying any templates in the former with the template definitions in the latter. The specification of the html_template_file property is only necessary if an applications needs to alter one or more SDK HTML template definitions.

EXAMPLE:

html_template_file=${my_context_path}\
WEB-INF/my_application_html_element_definitions.txt
jdbc_driver

Class name of the jdbc driver to use for the database connection. This is primarily used for developer testing purposes.

EXAMPLE:

jdbc_driver=com.microsoft.sqlserver.jdbc.SQLServerDriver
logging_config_file

This property instructs the AgencyPortal framework to defer all logging configuration to the JDK.

EXAMPLE:

logging_config_file=${my_context_path}WEB-INF/logging.properties
my_portal_app

Keyword used for the portal URL.

EXAMPLE:

my_portal_app=http://localhost:8105/AgencyPortal
my_portal_app_frontservlet

Keyword used for the FrontServlet URL.

EXAMPLE:

my_portal_app_frontservlet=${my_portal_app}/FrontServlet
option_list_providers

Provides two option list readers by default. One reads xml, the other reads options from a database table.

EXAMPLE:

option_list_providers=\
com.agencyport.html.optiongen.DefaultXMLOptionProvider;\
com.agencyport.dynamiclists.DynamicListProvider
PAS_URL

Path to the servlets to call to get the policy image.

EXAMPLE:

PAS_URL=${my_portal_app}/servlet/PolicyAdminSimultator
PERFORMANCE_RESULTS_FILENAME

If the performance system is turned on (by presence of a log file name), the framework writes a number of performance metrics to the output file.

EXAMPLE:

PERFORMANCE_RESULTS_FILENAME=${output_dir}perflogs/performance.log
PERFORMANCE_RESULTS_TRUNCATE_ON_STARTUP

A new performance log file is created on server startup when this property is set to YES. The default value is NO.

EXAMPLE:

PERFORMANCE_RESULTS_TRUNCATE_ON_STARTUP=YES
POLICY_IMAGE_CLASS

Class that is responsible for retrieval of policy image to initiate the translation.

EXAMPLE:

POLICY_IMAGE_CLASS=com.agencyport.policyadmin.DemoPolicyImageRetriever
print_request_parameters_to_system_out

Defines whether the application should print all the HTML form fields out to the command window. The value is usually set to true during development and false otherwise.

EXAMPLE:

print_request_parameters_to_system_out=YES
print_session_attributes_to_system_out

Defines whether the application should print all the session attributes out to the command window. The value is usually set to true during development and false otherwise.

EXAMPLE:

print_session_attributes_to_system_out=YES
print_transaction_definition_file

Indicates whether the transaction definition files have to log.

EXAMPLE:

print_transaction_definition_file=true
SAVE_AND_EXIT_NEXT_PAGE

This entry defines where the application will "go" when the framework standard Return button is selected (in the HTML page).

EXAMPLE:

SAVE_AND_EXIT_NEXT_PAGE=/DisplayWorkInProgress
show_client_details

Show client information on the page an in the javascript objects. This includes role, login id and id. The value is false by default.

EXAMPLE:

show_client_details=true
show_extended_error_detail

Controls whether to display extended error details on the navigation error jsp (APErrorJSP.jsp). The value is false by default.

EXAMPLE:

show_extended_error_detail=true
system_generated_id_attribute_type

This property dictates the use of globally or document unique ids. If this property is missing or is equal to the value 'guid,' then a 36 byte globally unique id generator engages. If the value is 'docuid,' then a document unique sequence number generates.

The value generates in the following format: Nnnnnnnn (where N is the character N and nnnnnnn is a sequence number unique within the scope of both the current and original documents maintained by APDataCollection).

EXAMPLE:

system_generated_id_attribute_type=docuid
transaction_definition_file_dump_directory

Provides the location of the directory where to stage the transaction files.

EXAMPLE:

transaction_definition_file_dump_directory=${output_dir}transaction_dump/
transaction_file_manager_mode

Indicates whether the product artifacts are reloaded by the server when changed. Possible values for this property are dynamic and cache.

EXAMPLE:

transaction_file_manager_mode=cache
transactionName.root_command_class_name

This tells the system where to find the user tailored servlets for a transaction.

root_command_class_name is a constant, transactionName must match a transaction name (persauto in the above example), packageName is the Java package where any custom classes reside and .prefix (optional) is used to form a common prefix for all the custom classes.

EXAMPLE:

personalAuto.root_command_class_name=com.agencyport.servlets.persauto.CMD
work_item_record_lock_time_in_seconds

Number of seconds a work item is locked after a user is no longer making changes to a work item.If a user closes their browser window while in a work item, the work item is locked from the last update, plus number of seconds configured using this property. The default value is 600 (10 minutes) and blank; 0 (zero) or a negative number turns work item locking off, which is not recommended.

EXAMPLE:

work_item_record_lock_time_in_seconds=600
work_item.command_impl_package

If this property is set, then the package name configured has precedence over the default package for the command handler class.

EXAMPLE:

work_item.command_impl_package= com.agencyport.workitem.command
WORKLIST_BUFFER_SIZE

Sets the number of results that are buffered on the client side. If not specified, the value defaults to 250.

EXAMPLE:

WORKLIST_BUFFER_SIZE=250
WORKLIST_INITIAL_BUFFER_SIZE

The number of rows that are buggered on initial display of a work list. The default is 100. Setting it to a relatively small number reduces the initial required loading time.

EXAMPLE:

WORKLIST_INITIAL_BUFFER_SIZE=100
WORKLIST_NUMBER_OF_VISIBLE_ROWS

The number of rows that show up on the work list. This value defaults to 10.

EXAMPLE:

WORKLIST_NUMBER_OF_VISIBLE_ROWS=10
work_list.case_insensitive_search

Work list searches are case insensitive. Used for databases that are set to case sensitive. The value defaults to false.

EXAMPLE:

work_list.case_insensitive_search=true
