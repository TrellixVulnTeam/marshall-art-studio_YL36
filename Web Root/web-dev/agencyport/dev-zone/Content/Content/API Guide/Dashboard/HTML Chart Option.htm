HTML Chart Option

AgencyPortal offers an HTML chart option, which does not use Adobe Flash, but the Dojotoolkit chart library to render charts. For more information on Dojotoolkit charting, visit the Dojo Charting website.

By default, HTML-based charts are used to render all charts. To change the charts to use AmCharts, you must set the following property in framework.properties:

Dashboard.use_amcharts=true

Then, configure the report specific settings for HTML reports. To change the path of stored report settings, set the dashboard.report_settings_path property in framework.properties. This allows for you to change or remove any properties. You can also add additional properties not defined, if required.

Changing the Report Settings Location

The following is an example property setting to change the report settings location:

EXAMPLE:

dashboard.report_settings_path=${my_context_path}WEB-INF/my_report_settings/html
  • Default for HTML charts (from jar file): dashboard_report_settings/html
  • Default for AmCharts (from jar file): dashboard_report_settings/amchart

NOTE:

The standard report properties are in the dashboard.jar for reports delivered with the product.

Common Chart Settings

The following are common chart settings:

Title Title for the report driven from a resource bundle
Plot Determines the type of report
Type Type of chart (pie, stacked columns, lines and grid)
Theme

Determines colors and style for reports. You can change these settings by creating a custom theme or selecting a theme that comes from the dojotoolkit.

  • DojoX Charting Theme Preview (built-in theme options)
  • Dojo Chart Theming (creating a custom theme)

TIP:

If you are going to create a custom theme, make sure you read the Basic Custom Theme: SitePen section in the Dojo Chart Theming link. After you create your custom theme, you must add it to your apdashboard.jsp.

EXAMPLE:

<script type="text/javascript" src="<%=jspHelper.getWebContentPrefix()%>javascript/apdashboard/html/apdashboard.js?<%=versionNumber%>"></script>
Legend Determines whether a legend displays for the chart

Example Pie Chart Settings

The following example is located in apdashboard.jar/dashboard_report_settings/html_report_100_settings.xml:

EXAMPLE:

<settings>
	<title><![CDATA[${rb:dashboard:title.report101}}]]></title>
	<plot>
		<type>StackedColumns</type>
		<gap>2</gap>
	</plot>
	<theme>dojox.charting.themes.ThreeD</theme>
	<legend>true</legend>
	<horizontalLegend>false</horizontalLegend>
	<tooltip><![CDATA[${rb:dashboard:balloonText.html.report101}]]></tooltip>
	<xAxis>
		<majorTicks>true</majorTicks> 
		<minorTicks>true</minorTicks>
		<majorLabels>true</majorLabels>
		<min>0</min> 
		<rotation>30</rotation>
	</xAxis>
	<yAxis>
		<min>0</min>
		<vertical>true</vertical>
		<minorTicks>false</minorTicks>
		<microTicks>false</microTicks>
		<majorTicks>false</majorTicks>
	</yAxis>
</settings>

XML Key:

<gap>
 gap between stacked columns 
<majorTicks>, <minorTicks>
 show tick marks for points on xAxis 
<rotation>
 rotation of labels on xAxis 
<vertical>
 determines whether yAxis is vertical

Example Line Chart

The following example is located in apdashboard.jar/dashboard_report_settings/html/report_102_settings.xml:

EXAMPLE:

<settings>
	<title><![CDATA[${rb:dashboard:title.report102}}]]></title>
	<plot>
		<type>Lines</type>
		<markers>true</markers>
	</plot>
	<plot>
		<type>Grid</type>
		<hMajorLines>true</hMajorLines>
        <hMinorLines>true</hMinorLines>
        <vMajorLines>true</vMajorLines>
        <vMinorLines>true</vMinorLines>
	</plot>
	<theme>dojox.charting.themes.ThreeD</theme>
	<legend>true</legend>
	<horizontalLegend>false</horizontalLegend>
	<xAxis>
		<minorTicks>true</minorTicks> 
		<min>0</min> 
		<rotation>30</rotation>
	</xAxis>
	<yAxis>
		<min>0</min>
		<vertical>true</vertical>
		<minorTicks>true</minorTicks>
		<microTicks>false</microTicks>
		<majorTicks>true</majorTicks>
	</yAxis>
</settings>

Grid plot type allows for grid lines on line chart. This is optional.
