Registering the ACORD Simplified Model with AXE

The following is a sample of registering the ACORD simplified model with AXE:

  • Assume that you have placed the schema extensions to simplify ACORD for your application in an XSD file whose file name abc_simplified_acord.xsd and is deployed in a directory under WEB-INF called shared/schemas.
  • Assume that the LOB for your application is BOP.

The following AXE configuration follows:

DATA_SCHEMAS+=${my_context_path}WEB- 
INF/shared/schemas/abc_simplified_acord.xsd[Element:BOPPolicyQuoteInqRq]

The next time your server boots, your ACORD simplified data model is active.

We assume that the precondition data store shares the same schema with that of the main work item XML. If it does not, then you will need to take this into account.
