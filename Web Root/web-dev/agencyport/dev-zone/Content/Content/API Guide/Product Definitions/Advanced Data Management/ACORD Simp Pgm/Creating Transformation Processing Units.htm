Creating Transformation Processing Units

From looking at the transformer.xsd XML schema file, notice that the transformer can comprise of one or more steps. Each step performs a discreet set of transformations and internally invokes an implementation of the com.agencyport.domXML.transform.IAPDataCollectionTransformer interface.

You can write transformation steps as XSLT or Java. The product has its own wrappers around the XSLT units, conforming to this interface so there is no need for you to wrap the XLST. If you package your transformation logical units as Java, you must implement the aforementioned interface.

The transformation examples in the Sample Transformers section supports the sample BOP simplified data model previously referred to in this documentation.
