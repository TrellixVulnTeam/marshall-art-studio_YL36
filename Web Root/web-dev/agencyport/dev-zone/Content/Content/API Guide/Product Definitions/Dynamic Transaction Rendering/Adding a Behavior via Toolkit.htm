Adding a Behavior via Toolkit

Perform the following to add a behavior via Toolkit:

  1. Click on the Behaviors file.
  2. Click Behaviors in the editor.
  3. Click Add Behavior at the top of the editor.
  4. Click the Options tab.
  5. Change the Action field to indicate the action for the behavior. Valid values are:
    • Include - used to conditional display data.
    • Exclude - conditionally hide data.
    • Alter - many purposes
  6. Click the Page Entities tab to define the field that will display on the page (or hide/alter, depending on the action type) based on a certain condition.
  7. Click the Where tab to define the condition that will trigger the behavior. If there are multiple triggers, click the Add precondition to 'where' clause button to add a new row.

    EXAMPLE:

    If the driver's license date is after 1990-01-01, the field defined in the page entities section displays.


    The Actions tab sets the value(s) that must be selected or entered for the behavior to get triggered. If Is In or Is Not In is selected for test, enter the value in single quotes (e.g., 'MA'). You must separate multiple values by commas (e.g., 'MA', 'CT', 'NH'). If a date operator is selected for test, enter the date in single quotes using the format 'YYYY-MM-DD' (.e.g, '2010-01-01'). Refer to Behaviors: Built-in Preconditions and Operators for more information on built-in preconditions and a list of possible operations.


    NOTE:

    The Precondition Returns field on the Where tab allows you to indicate whether a precondition should be set based, cound based or count of a set.
