Behaviors: Built-in Preconditions and Operators

The following is a list of built-in preconditions:

Available built-in precondition Description
hasDataChanged Returns true if data has changed on the current page. This precondition can and should be only applied to connectors and instructions and only makes sense on the process side of things.
hasPermission Returns all of the names of the permissions for all of the permissions that the user has been granted.
inIPDTRMode Returns true only when an Intrapage DTR AJAX call is in context.
inPreValidationMode Returns true if prevalidation is currently active. This could be on if a TVR is also running.
inRosterEditMode Returns true on the display side and not on the process side when a roster is being edited.
inUploadWriterMode Returns true if in an upload writer context.
isCurrentPageVisited Returns true if the current page has been visited.
isDataAvailable Returns true if data is available on the current page.
isUserInRole Returns true if the current user is in the role group namepates in.
locale_country Returns the country for the locale currently in context.
locale_language Returns the language for the locale currently in context.
locale_variant Returns the variant for the locale currently in context.
recordLockStatus Returns the record lock status. Possible values are LOCKED_BY_THIS_USER, LOCKED_BY_OTHER_USER, LOCKING_NOT_ACTIVE or UNKNOWN.
statusCode Returns the status mnemonic of the status code for the current work item.

The following is a list of precondition operations:

Available Operators Description XML Example
Is Equal To Equals the value specified. 
 <preCondition name="isUserInRole">agent</preCondition>
Is Not Equal To Not Equal to the value specified. 
 <preCondition name="isUserInRole">!agent</preCondition>
Is In Is in one of the values specified. 
 <preCondition name="isUserInRole">in('agent','agentcl','agentpl')</preCondition>
Is Not In Is not in one of the values specified. 
 <preCondition name="isUserInRole">!in('agent','agentcl','agentpl')</preCondition>
Is Less Than Is less than the value specified. 
 <preCondition name="NumberOfIncidents">lt(2)</preCondition>
Is Not Less Than Is not less than the value specified. 
 <preCondition name="NumberOfIncidents">!lt(2)</preCondition>
Is Greater Than Is greater than the value specified. 
 <preCondition name="NumberOfIncidents">gt(2)</preCondition>
Is Not Greater Than Is not greater than the value specified. 
 <preCondition name="NumberOfIncidents">!gt(2)</preCondition>
Is After the Date Is after the date specified. 
 <preCondition name="PolicyEffectiveDt">DateAfter('2008-01-01')</preCondition>
Is Not After the Date Is not after the date specified. 
 <preCondition name="PolicyEffectiveDt">!DateAfter('2008-01-01')</preCondition>
Is Before the Date Is before the date specified. 
 <preCondition name="PolicyEffectiveDt">DateBefore('2008-01-01')</preCondition>
Is Not Before the Date Is not before the date specified. 
 <preCondition name="PolicyEffectiveDt">!DateBefore('2008-01-01')</preCondition>
