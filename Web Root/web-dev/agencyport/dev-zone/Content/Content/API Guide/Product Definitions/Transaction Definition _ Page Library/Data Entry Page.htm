Defining/Editing Data Entry Page Properties

A data entry page is a type of page used to collect basic, non-repeating data.

EXAMPLE:

General Information is collected on a data entry page.

Perform the following to define/edit data entry page properties via toolkit:

Select a new or existing page in the editor. The screen layout displays in the palette.

Complete the following required fields for the page under the Options tab:

Title - the display name for the page.

ID - the unique ID within the page library.

Field Validations - Indicates how field validations on the page run. Valid values are:

Enabled - always run on both client and server side.

Disabled - do not run on either client or server side.

Client side disabled - only run on server side.

Server side disabled - only run on client side.

Click Advanced tab.

NOTE:

The Impacted By and Used By tabs are read-only. Click each tab to view information on the following:

Impacted By lists behaviors used by the page. Click on a behavior to view behavior details.

Used By list of transactions using the page. Click on one of the transactions to go directly to the transaction page.

Menu Display - Indicates when the page displays in the left navigation of the application. Valid values are:

Slow Loader - Indicates whether a "please wait" lightbox displays when the page is processing. This is used on pages that are slow due to connections with rating engines or other third party services. Valid values are:

true - a "please wait" lightbox displays when the page is processing.

false - a "please wait" lightbox does not display when the page is processing.

Ignore Page Visited Check - Indicates whether to ignore the page visited check. Valid values are:

true - the page is disabled after it is submitted if the page has been visited in an application.

false - the page is enabled in the left navigation menu if the page has been visited in an application.

Prevalidate - determines whether fields are validated before an initial page displays in the application. Valid values are:

true - pages are validated before an initial page displays in the application.

false - pages are not validated before an initial page displays in the application.

Style Class - Enter a style class if you want to change the default style of the page.
