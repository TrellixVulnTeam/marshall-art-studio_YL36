Adding a View via Toolkit

Perform the following to add a view via toolkit:

Add a Standard View

  1. Navigate to your Views collection folder.
  2. Select Add Standard View at the top of the editor. Five types of standard views display:
    • Standard
    • Aggregate Exists Boolean - used with fields not stored in the data store when a boolean false is selected.
    • Remarks - used to store remarks with a reference to a foreign aggregate.
    • Date Format - used to store a date in the ACORD format (YYYY-MM-DD) and read the date back to the screen format.
    • Custom - coded by a developer when none of the provided views will satisfy the requirement.
    • An empty instruction set displays when adding a standard view. A menu button also displays at the top of the editor, which allows you to delete the new view.

Add a Field

  1. Click the Instruction Set legend. Two buttons display at the top of the editor, which allow you to manipulate the view:
    • Add a Field
    • Add a Delete Field
  2. Click Add a Field. There is a pattern for adding fields. The initial field to add is the path.
  3. Click the '(Click here to enter a value)' link and enter your valid ACORD XPath.
  4. Click Add a Field again. This add generates a format field. The sequence of alternating field types continues on any additional field adds. Delete fields does not follow this pattern and are added as single delete fields.
  5. Click the (Click here to enter a value) link. Enter your valid ACORD Xpath.
    Format fields are either literal, which require a value, or singleSpace, which do not require a value. The singleSpace format value field is disabled.
    After a field is added, you can delete it by click on the field in the editor. This action updates the Delete button to refer to the clicked field and enable deletion.
    The following example shows a functioning standard view and its representation in the portal's XML. 
    <Coverage>
					<CoverageCd>UNDSP</CoverageCd>
					<Limit>
					<FormatInteger>50000</FormatInteger>
					<LimitAppliesToCd>PerPerson</LimitAppliesToCd>
					</Limit>
					<Limit>
					<FormatInteger>100000</FormatInteger>
					<LimitAppliesToCd>PerAcc</LimitAppliesToCd>
					</Limit>
			</Coverage>

Add a Display View

Refer to the Add a Standard View procedure, except there is no capability to add a delete field.

Add a Mutually Exclusive View

The mutually exclusive view differs from a standard view. A mutually exclusive view requires at least two fieldsets to be present, neither of which can be deleted. Each fieldset contains a field 'If this value is found,' which corresponds to the searchId used by the view. There must be a character value in at least one of these fields for the view to function properly. These fields cannot be deleted and do not have a Delete button available.

The above view is represented in the xml in one of two ways, depending on the evaluation of the searchId:

  • If the / character is found, the resultant xml is: 
    <Coverage>
    <CoverageCd>UM</CoverageCd>
    <Limit>
		<FormatInteger>25000</FormatInteger>
		<LimitAppliesToCd>PerPerson</LimitAppliesToCd>
    </Limit>
    <Limit>
		<FormatInteger>50000</FormatInteger>
		<LimitAppliesToCd>PerAcc</LimitAppliesToCd>
    </Limit>
</Coverage>
  • If there is no character to be evaluated, then the resultant xml is:
    <Coverage>
    <CoverageCd>UMCSL</CoverageCd>
    <Limit>
		<FormatInteger>50000</FormatInteger>
    </Limit>
