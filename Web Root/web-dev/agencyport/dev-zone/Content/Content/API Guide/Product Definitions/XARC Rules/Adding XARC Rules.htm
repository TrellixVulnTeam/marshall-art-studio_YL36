Adding XARC Rules/Connectors

Connectors are the mechanism by which the framework integrates with third party services or the way to apply business specific logic. A common connector is one that triggers ArcRules.

Rules are only fired if a connector is created to trigger them.

Perform the following to add XARC rules/connectors via Toolkit:

  1. Click on a page from a page library. By default, sections of the page display in the editor.
  2. Click on the Connectors tab in the editor to create your connector.
  3. Select the Process Side option. Leave any default options in the palette as is.
  4. Select Add Connector in the editor and choose xARC. The new connector displays highlighted in blue.
  5. Enter the following required information in the Options tab:
    • Title - the name for the connector.
    • ID - an ID that is unique within the page library.
    • Description - describes what the connector does.
    • Suppress Message Display - if true, this suppresses any messages that are triggered by the connector.
    • Run Connector on Prevalidate - if true, this runs the connector on prevalidate (on page load).
  6. Enter the trigger's actions by clicking Add Triggering Action. Each action must have its own ID that must be unique within the page library. In the actions field, start type a few letters to get help for the actions available. A common option is ContinueSave.

    TIP:

    You can use either Continue and RosterContinue for a roster page. The list of available actions are Display, Continue, SaveAndExit, Save, Finish, ContinueSaveAndExit, ContinueSave, RosterDisplay, RosterAdd, RosterCopy, RosterEdit, RosterAddCopyEdit, RosterDelete, RosterContinue, RosterSaveAndExit, RosterContinueSaveAndExit and RosterAllExceptDelete.

  7. Select a Rule Collection or single Rule to enter all the rules that should run.
  8. Enter a unique ID if you are writing behaviors around this action.
  9. Click Add Rule if you are selecting single rules to run instead of a collection.
