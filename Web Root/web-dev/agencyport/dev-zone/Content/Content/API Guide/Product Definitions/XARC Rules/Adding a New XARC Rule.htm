Adding a New XARC Rule

Perform the following to add a new XARC rule via toolkit:

  1. Select a ruleset (collection) in the navigation window.
  2. Click the Add Rule action in the editor. The associated editor and palette for the new rule display.
    The rules editor has three sections:
    • Variables
    • Conditions
    • Messages

    A default title is assigned to the new rule.
  3. Enter the required field ID and title in the palette.
  4. Add variables (optional) if you want to create a rule with a message that uses a value from the application (e.g., "The limit of $50,000 entered is ineligible for this coverage") of if you are creating a rule that needs to count in a roster (e.g., "At least one location must be entered.").
    1. Perform one of the following:
      • Click Add on the far right side of the Variables section in the editor.
      • Click Delete at the top of the editor.
    2. Click the Options tab in the palette and enter the following required fields:
      • Name - name of the variable
      • Scope - determines where the variable will be available for use Valid values are:
        • Local - the variable is available for a specific rule
        • Rule File - the variable is available for all rules in the ruleset
        • Rule - more than one rule can use the variable name without conflict
      • Type - the variable type. Valid values are:
        • Field - if selected, additional drop-downs display in the palette too allow the user to drill down to the field variable by selecting the transaction, page, fieldset and field.
        • Value - if selected, enter a value.
        • Xpath - if selected, enter an Xpath.
        • Variable - if selected, select a variable (any that have been created, as well as "Today's Date").
        • SQL Query - if selected, enter the data source and SQL query path.
        • Expression - if selected, a new palette tab, Expression, displays, which requires the expression detail. This is used with more complex rules.

      The following is an example of the editor and palette for the Effective Date variable in the personal auto rule: "Effective Date cannot be more than a month in the future."

      TIP:

      For field operand type, the list of fields available for selection is limited to ones not hidden and that do not have custom view or special field helpers.
