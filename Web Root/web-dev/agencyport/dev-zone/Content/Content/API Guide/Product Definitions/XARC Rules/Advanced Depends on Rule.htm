Advanced Depends on Rule

Rule:

Condition1: Incident type of major violation in the last five years; Condition2:any other incidents in the last three years.

If the second condition is "No," it's a referral; if the second condition is "Yes," it's a decline.

Fields:

Accident Violation Type: PersPolicy.AccidentViolation.AccidentViolationRecordTypeCd

Accident Violation Date: PersPolicy.AccidentViolation.AccidentViolationDt

Approach:

Since condition one is common for both the referral rule and decline rule, create a separate rule for it and execute the other rules depending on its outcome.

  1. Incident type of major violation in the last five years.

    NOTE:

    There is no message added to this rule since it is not executed on its own.


  2. Any other incidents in the last three years display decline message. Add the dependency on the rule1 as "rule1=true."
  3. If no other incidents in the last three years, display referral message. This condition is impossible to write with where conditions because you need to display a message when the conditions are not met.
    The following is the problem with trying to do this with a where clause:

    where1: accident date is before today's date

    where2: accident date is after 3 year old date

    where3: accident type is not equal to ACCAF

    where4: count (accidents) = 0


    In case there are no accidents other than ACCAF in the last three years in the work item, the first three conditions are not met and the fourth condition is NOT executed. This causes the rule to fail and no message to display.
    This behavior is consistent with the design to generate a message only when the conditions are met. For you to display a message when the conditions did not meet, use the depends on feature and test whether the rule written for the else part of this requirement returns as false. If it returns false, display the message.
    dependsOn=”rule1=true;rule2=false”
