Repeating Aggregate Rule (Sample 1)

Rule:

At least one location should be controlling state. Display the following error message: "At least one location should be from controlling state."

Fields:

Location State: /PersAutoPolicyQuoteInqRq/Location/Addr/StateProvCd

Controlling State: /PersAutoPolicyQuoteInqRq/PersPolicy/ControllingStateProvCd

Approach:

Since rules generate messages when a set of conditions are met, the condition to be met is NOT (location state = controlling state).

TIP:

One rule of thumb to follow is if the rule is “at least one of” kind of a rule, then you don’t need to loop over the aggregates.

Steps to write this rule:

  1. Create a new rule and provide the mandatory information.
  2. Add a condition and, for the left operand, choose the State field on the location page.
  3. For the Operator, choose "NOT=".
  4. For the right operand, choose the controlling state field. If it is a hidden field, special field or field with a custom view, and is not available for selection, use the xpath type and enter the xpath for that field.
  5. Add a message.
