Rule Level Variables

Rule:

Check if the policy is a multi-state policy.

Fields:

Location State: Location.Addr.StateProvCd

Approach:

Multi-state check is generally a pre-requisite for other rules. You can take the state of the first location as a reference state and check whether a location exists with the other than the reference state.

Steps to write this rule:

  1. Add a rule with Page Source set to Location.
  2. Create a rule scope variable named firstLocationState. Use the field type of xpath and enter the xpath.
    /PersAutoPolicyQuoteInqRq/Location[position()=1]/Addr/StateProvCd.
    This xpath gets the first state.
  3. Add a condition to eliminate all locations with state NOT-firstLocationState. If there are locations with state other than the first state, a multi-state situation exists.
  4. Add the message to display with appropriate severity level.
