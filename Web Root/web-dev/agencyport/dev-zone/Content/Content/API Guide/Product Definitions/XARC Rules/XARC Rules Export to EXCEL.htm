XARC Rules Export to EXCEL

An export to Excel function for XARC rules within toolkit has been available since version 4.2. This feature allows you to export the rule inventory for a specific product into an Excel spreadsheet. This feature was developed in response to client and project requests for a function to export rule sets into a readable/printable format.

User Interface

Users with access to the toolkit can go to the specific product in which they want to import rules.

  1. Click on the product's Rules folder in the navigation menu. An Export to Excel button displays.
  2. Click the Export to Excel button. A pop-up window displays to select whether to open or save the file.
    • The default format of the spreadsheet created is version .xlsx.
    • The exported spreadsheet contains a tab for each rule file that exists in the rules folder. Each rule file tab contains data for the following columns:
      • Name - the id of the rule.
      • Title - the title of the rule.
      • Description - the business reason for the rule.
      • Used By - the pages that utilize the rule.
      • Author - the individual that created the rule via the toolkit.
      • Date Created - the date in which the rule was created via the toolkit.
      • Message - the message displays when triggered.
      • Message Class - the role for which the rule applies. This field populates when there are multiple rule messages.
      • Severity - the type of rule message: informational, warning or error.

      NOTE:

      If there is more than one rule message for a given rule, a new column is created for those messages along with the message class and severity.
