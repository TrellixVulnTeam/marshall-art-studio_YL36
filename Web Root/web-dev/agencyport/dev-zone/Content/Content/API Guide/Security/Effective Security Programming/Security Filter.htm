Implementing the Security Filter

The security filter is a J2EE servlet filter implemented specifically for the purposes of making authorization assertions on an HTTP request in the context of the security profile of the current user. When an authorization assertion fails, an exception is raised and the delegation to any downstream filters or target servlet is aborted. The security filter is designed to guard against a user tampering or contriving URL and their associated parameters. This practice is referred to as URL spoofing.
