Security Runtime Release Versioning

This section outlines changes to the core security library since AgencyPortal 4.0. Depending on the version of the SDK that your application is based on, these details may impact the specific techniques outlined in the step-by-step guide:

  • The com.agencyport.security.credentials.CookieAgent class was promoted to the core security library. It was originally packaged as part of the security reference library. There was no change to the public interface.
  • The com.agencyport.security.credentials.CredentialsAgency class in the core security library became a concrete, instantiable class. All methods previous found in the com.agencyport.security.credentials.BuiltinCredentialsAgent class from the security reference library were promoted to this class to make the class in the security reference a shell of its former self.
  • Much of the functionality that was originally part of the com.agencyport.security.profile.builtin.BuiltinSecurityProfileManager class in the security reference component was either promoted up to the com.agencyport.security.profile.SecurityProfileManager or to a new brand class in apwebapp.jar called com.agencyport.security.profile.builtin.PortalSecurityProfileManager. Depending on which version of the SDK you are using, this will impact which base profile manager class you will extend from.
