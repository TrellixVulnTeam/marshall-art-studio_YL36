Creating a New Assistant

The following is an example of you can create a new, custom Work Item Assistant via toolkit and add it to your desired product's transaction.

  1. Open the folder for the product you want to work with. In this example, we are using Commercial Umbrella. Commercial Umbrella, like all other products, uses a list of resources (transactions, page libraries, etc).
  2. Click on Work Item Assistant.

    At the top of the editor, two buttons display: Add and Import.
  3. Click the Add button. A create work item assistant pop-up displays.
  4. Enter values in the following fields and click Create:
    • Field ID - A unique id for your assistant file. You cannot use this id anywhere else within the product (Commercial Umbrella). DO not use spaces, >, <, /, \ or " characters.
    • Assistant Title - The file name for your assistant. The assistant title is only used inside the toolkit for you to identify the file. In this example, we're using Assistant for CUMBR New Business.
    • Description - An optional description for the assistant.
    • Assistant ID - A unique id used for transactions to identify the assistant file. This must be unique among all products. Do not use paces, >, <, /, \ or " characters.

    An empty assistant editor screen displays. The title of your assistant displays int he navigation menu on the left and in the top right corner of the editor.
  5. Click the Add Section button at the top of the editor.
  6. Choose Comments from the drop-down. The New Comments section displays in the editor
  7. Change the Label field to Agent Comments to label the assistant section. This is the label that displays in the assistant inside the portal. The Section Type and ID fields are read only, which lets you know the type and ID for the section.
  8. Edit the roles for this section. We want agents and underwriters to have the ability to use this section. Add roles for agent and underwriter. These role names are part of the Agencyportal security model. Your role names may vary based on your project implementation.
  9. Click New Role in the editor and type in the role name.
  10. Click Save at the top of the editor.
  11. Add a file attachments section. Click Add Section at the top of the editor and choose File Attachments. Your New File Attachments section displays in the editor.
  12. Change the label to CUMBR File Attachments. Leave the list of Role Names blank (this means all of your users have access to the CUMBR File Attachments section).
  13. Click on the Document Type tab in the editor.

    The File Attachments section on the assistant screen contains a browse field for the user to upload the attachment and a Document Type drop-down selection field. This drop-down field allows the user to specify the type of document they are attaching to the workflow.
  14. Define the option available to the user in the drop-down field.
  15. Click New Value.
  16. Click on the Collection drop-down and select the codeListRef.xml list collection. If you do not already have a list collection for this exercise, you must create a new one.
  17. Click on the List drop-down and select the fileAttachmentType list.
  18. Click Save at the top of the editor.
  19. Rearrange the order of your assistant sections. Click on CUMBR File Attachments in the editor and click Move. A pop up displays.
  20. Select Before as the Move value and Agent Comments as the Sections value, and click Apply. CUMBR File Attachments is now the first section.
  21. Click Save.
  22. Add the new assistant to a transaction.
  23. Click the transaction resource under Commercial Umbrella in the navigation menu.
  24. Open the New Commercial Umbrella Application transaction file.
  25. Scroll down to the bottom of the palette and select Assistant for CUMBR New Business in the Work Item Assistant drop-down.
  26. Save the change.
  27. Start a new Commercial Umbrella application in the portal.
  28. Fill out the first page and click Continue. The new assistant displays.
