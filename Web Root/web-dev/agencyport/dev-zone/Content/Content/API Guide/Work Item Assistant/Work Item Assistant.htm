Work Item Assistant

Work Item Assistant was developed in response to the increasing demand for collaboration technology within the portal. We have learned firsthand from agents that a high degree of collaboration between agents and a variety of parties at an insurance company are often required to issue new insurance policies. Numerous email, fax and telephone exchanges are required for business to take place between parties, increasing the operating expenses associated with the process and negatively impacting the purchasing experience for the insurance buyer.

Work Item Assistant t helps alleviate much of the back and forth faxing, emailing and telephone calls by allowing increased communication within the portal. The assistant allows agents and insurance carrier employees to communicate real time with in a quite, upload file attachments at any point within the transaction and receive email notifications for specific events.

Interface

Work Item Assistant becomes available only after the user saves or continues from the first page in the work item. At that point, the transaction is created in the database. The assistant displays on the right hand side of the portal when a user opens a work item/quote. The assistant is an accordion that you can configure to contain multiple accordion sections, including the following:

  • comments
  • file attachments
  • other active users

The assistant allows you to add comments and file attachments on any page within a transaction, as well as allow users to see that other users are working within the same work item. This element of the feature allows for nearly real-time communication among users that are in the same work item.

IMPORTANT:

The following are a couple of important points about the interface:

You can turn on/off Work Item Assistant at the transaction level.

Each section is available to all users by default; however, you can configure this availability by user role and role group (such as agent or underwriter).

A count displays in each section to indicate how many comments, file attachments and active users exist for the transaction. You can expand each section to view the actual comments, file attachments and active user usernames.

Comments, file attachments and active user usernames are added to the assistant within 15 seconds of their entry (i.e., it may take up to 15 seconds for a comment to display, but no longer).

You can configure multiple sections of the same type (e.g., a given assistant can have multiple comment sections).

User Roles and Role Groups

You can configure the accordion sections within the Work Item Assistant to allow some user roles more access and permissions than others. You can also put restrictions on a section so that only a specific set of users can view and access that section.

Users associated to a specific role or role group who are not defined for a specific section cannot view that section in the application.

EXAMPLE:

The assistant can be configured so that internal comments can only be viewed by underwriters and external comments can be viewed by underwriters and agents.

Comments and internal comments sections are included; there can be multiple comments and file attachment accordion sections with unique names and user/role groups set up.

Comments

Click on the Comments accordion section header to view any comments entered for the work item. To leave a comment, enter text in the text box and click the Comment button. Comments entered display within 15 seconds; you cannot delete comments.

File Attachments

Users have the ability to add file attachments to any page within the transaction by expanding a File Attachment accordion section, browsing for the file and attaching it to the assistant. File attachments display within 15 sections of uploading. You can open any file attachment within the assistant by double-clicking on the file icon.

NOTE:

The file attachment section display is slightly different in a Chrome browser; other browsers display a button labeled Browse and Chrome displays the button as Choose File.

A recent addition to the File Attachment section is the ability to select Document Type. IF the list of types needs to differ by line of business, you can configure Work item Assistant for each product to meet this need. You can write custom rules to validate whether a specific document type exists if certain types are required for your business. The display of the Document Type field is optional. If the field displays, a document type is required for selection.

Other Active Users

If you are in a work item that another user is currently viewing, the assistant lets you know of the other user's presence.

  • When other users have the work item open, the username of each other active user displays in the accordion section.
  • If there are no other active users in the work item, the following text displays when you expand the Other Active Users section: No other active users.

Email-Based Notifications

Functionality is provided to streamline the process to send email notifications to specific users or user groups when:

  • the status of a work item changes
  • a comment is made
  • a file attachment is added

With this functionality, users do not have to be within a work item or even logged into the portal to be notified of activity on a quote

The email notifications contain a link that brings users to the associated work item.
