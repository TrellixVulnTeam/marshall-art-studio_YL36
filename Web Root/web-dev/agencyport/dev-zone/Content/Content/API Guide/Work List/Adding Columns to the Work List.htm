Adding Columns to the Work List

There are substitutions when a column need to be added to a given work list. In the following example, an existing column from the work_item table is added to the work list for administrators.

NOTE:

In this instance, the work item table already contains the expiration_date column. To add a brand new column to a work list, following the directions in the Work Item Management Example topic, then following the directions below.
  1. Change work list query to include expiration_date form work_item table. 
         UPDATE work_list SET work_list_sql = 'SELECT
						work_item.work_item_id "Work Item #",
						work_item.lob "LOB",
						work_item.entity_name "Name",
						work_item.last_update_time "Updated", status_codes.after_title "Status",
						work_item.effective_date "Effective",
						work_item.premium "Premium",
						work_item.creation_time "Created",
						work_item.upload_flag "Upload Flag",
						work_item.transaction_type "Transaction Type",
						work_item.expiration_date "Expiration Date"
						FROM
						${db_table_prefix}work_item work_item,
						${db_table_prefix}status_codes status_codes
						WHERE
						work_item.status = status_codes.status_code_value
						AND
						commit_flag = 1
			ORDER BY 4 DESC' WHERE work_list_id = 100;
  2. Create the new column in the work_list_column table for the new expiration_date work list column. 
         INSERT INTO work_list_column(
						work_list_column_id,
						work_list_id, table_alias,
						column_name,
						column_alias,
						format, type,
						column_alignment,
						filter_enabled, column_enabled,
						sort_enabled,
						column_number,
						identity_column)
						VALUES (
						111,
						100,
						'work_item',
						'expiration_date',
						'Expiration Date',
						'MM-dd-yyyy','date',
						'text-align:right;',
						1,
						1,
						1,
						11,
			0);
  3. Add the item to the acordtoworkitemmap.xml file if the data needs to be written from ACORD XML to the work_item table.

    NOTE:

    In this case, the expiration_date column is already added. If creating a brand new column, an entry needs to be added here if mapping the data from ACORD XML to the work_item table.


        <?xml version="1.0" encoding="UTF-8"?>
						<!-- id is the transaction.@mappingId -->
						<acordtoworkitemmap xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
						xsi:noNamespaceSchemaLocation="http://reference.agencyport.com/schemas/4.0/acordtoworkitemmap/acordtoworkitemmap.xsd"
						id="workersComp">
						<mapEntry>
						<xpath valueType="string">InsuredOrPrincipal.GeneralPartyInfo.NameInfo.CommlName.CommercialName</xpath>
						<workitemproperty>ENTITY_NAME</workitemproperty>
						</mapEntry>
						<mapEntry>
						<xpath valueType="date">CommlPolicy.ContractTerm.EffectiveDt</xpath>
						<workitemproperty>EFFECTIVE_DATE</workitemproperty>
						</mapEntry>
						<mapEntry>
						<xpath valueType="date">CommlPolicy.ContractTerm.ExpirationDt</xpath>
						<workitemproperty>EXPIRATION_DATE</workitemproperty>
						</mapEntry>
						<mapEntry>
						<xpath valueType="string">CommlPolicy.LOBCd</xpath>
						<workitemproperty>LOB</workitemproperty>
						</mapEntry>
			</acordtoworkitemmap>
