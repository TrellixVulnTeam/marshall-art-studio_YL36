Work List

[NEED 5.0 SAMPLE]

A work list is used to present a large amount of data to a user in a tubular format. Optionally, you can configure a work list to allow a user to take action on member items within the list. The quintessential example of a work list in s the work in progress queue. Work lists are also used on the dashboard to present the audit events.

  • A user can take actions, such as opening or copying work item, for a work in progress list.
  • A user can only view the items as a list for an audit table list.

Work list supports both of these scenarios.

Work list configuration allows the implementer to specify columns within the list and actions (if any) associated with the member items. Implementing a work list provides all the conveniences of the base implementation, including scrolling, sorting and filtering list items via Ajax. While work lists can co-exist with other items on a page, such as in the aforementioned dashboard scenario, they are not designed to be used within a transaction.

TIP:

Need to implement a WIP? Work lists are what you're looking for.

Work List Feature

The work list provides information on transactions that were previously entered in the application. You can configure these lists to display work items based on user type. An underwriter may have the ability to view all work items entered; whereas an agent can only see the work items he or she created.

Work List Columns

The following columns come as part of the work list:

WIP Column Description
Work Item # The work item number is a number systematically assigned when a user continues from the first page of the application.
LOB This column is the line of business.
Name The name of the applicant or insured is captured in this column.
Updated This is the time and date that the work item was last updated.
Status The status of the work item. Some examples are: In Progress, Referred, Approved, Rejected, Declined.
Premium When a work item is submitted, a premium may be returned. This column displays this amount.
Transaction Type Examples include: Quick Quote, New Business, Endorsement, Renewal

Filtering

Along with sorting, you can perform the following filters depending on the column:

Filter Actions for Columns in the Worklist Description
Equals This action is used for numeric fields and brings back the exact match entered.
Is greater than For numeric fields, this returns all values greater than the value entered.
Is less than For numeric fields, this returns all values greater than the value entered.
Is This action brings back the exact match entered.
Contains Brings back all criteria that matches.
On For date fields, this brings back the matches prior to the date entered.
Before For date fields, this brings back the matches prior to the date entered.
After For date fields, this brings back the matches after the date entered.

Actions on Specific Work Items in the Work List

You can perform the following actions on a specific work item in a work list:

Action

Description
Open Opens up the specific work item that you would like to view or edit.
Assign This action allows the work item to be assigned to a different user.
Copy A work item that was previously entered can be copied into a new work item.
Delete A work item can be deleted with this action.
