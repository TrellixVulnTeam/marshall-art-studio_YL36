Alternate Steps: Consuming the Archetype from an FTP Site

If you don't have access to our Nexus site, we publish the contents of our repository to an FTP site. At this time, Maven does not support archetype downloading from an FTP site, so the following steps are a littler different than those listed in the Setting Up Your Portal 5 Project procedure.

NOTE:

Maven holds credential details and other user specific configuration in the settings.xml files. Refer to the Setting Details topic on the Sonatype site for more information.

Every developer is responsible for setting up their settings.xml file appropriately with their credentials.

Perform the following to consume the archetype from an FTP site:

  1. Create a folder on your machine to house the archetype files.

    EXAMPLE:

    c:\tools\archetype\

  2. Log into the FTP site (using FileZilla or some other tool) and copy the archetype-catalog.xml file from the FTP site to the new folder you just created.
    The catalog file is located at sftp.agencyport.com/repositories/releases/archetype-catalog.xml.
  3. Open up the archetype-catalog.xml file on your machine with a text editor. The archetype elements in there have a child element named repository, which is currently pointing at the FTP site. You need to update this element to point it back into the folder on your machine in the location where you have been working.

    EXAMPLE:

    <repository>file://C:/tools/archetype/ </repository>

  4. Create the following folder structure under the stage folder:

    com\agencyport\sdk

  5. Copy the portal-archetype folder from the FTP (located at sftp.agencyport.com/repositories/releases/com/Agencypot/sdk/portal-archetype) to the sdk folder created in step 4.
  6. Open a command prompt and run the following MVN archetype:generate command (substitute in the correct path from your filesystem, if necessary):

    7. mvn archetype:generate -DarchetypeCatalog=file://C:\tools\archtype\test\archetype-catalog.xml

  7. Follow the command prompt instructions above and create your project from the archetype.
  8. Open the POM file. Towards the bottom of the file, there are some commented out settings you can use to point at the FTP location to resolve the AP SDK dependencies.
  9. Uncomment the FTP deployment settings and comment out or delete the Nexus deployment settings.
  10. Go to USER_HOME/ .m2/settings.xml and add the FTP credentials (refer to pom.xml) to the settings.xml file under the <servers> aggregate.
