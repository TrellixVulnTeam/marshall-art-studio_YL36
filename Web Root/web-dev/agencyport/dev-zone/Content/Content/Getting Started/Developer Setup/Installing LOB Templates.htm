Installing LOB Templates

Template dependencies are not packaged into the archetype. This is to allow developers to selectively include the line of business (LOB) templates of their choice into the environments. Installing a template is a one-time process. One or more template zip files are pulled down from the repository and then extracted into your project.

Perform the following to install and include a LOB template(s):

  1. Reference the lob-install profile that came with your pom.xml. The maven-dependency-plugin lists an artifactItem dependency on commlauto and homeowners by default. You can add/remove other LOBs if needed. Browse the Nexus server to see the available templates.
    2.                <artifactItems>    <!-- you may add multiple LOB's by adding additional artifactItem --> 
                    <artifactItem> <!-- Please check the AP Maven Repo for available artifacts  --> 
                         <groupId>com.agencyport.sdk</groupId>
                         <artifactId>aptemplates</artifactId>
                         <version>5.0-SNAPSHOT</version>  <!-- Version Name --> 
                         <classifier>commlauto</classifier> <!-- LOB Name --> 
                         <type>zip</type>
                         <overWrite>false</overWrite>
                         <outputDirectory>${project.basedir}/</outputDirectory>
                    </artifactItem>
               </artifactItems>
  2. Run the following command from the root of your project:

    3. mvn -P lob-install dependency:unpack resources:copy-resources clean

    NOTE:

    As part of this process, a lob-markers folder generates. The meta-data within this folder prevents a template, which has already been installed, from being overwritten if the command is run on subsequent occasions. As such, you should commit the lob-markers directory to your SCM system.
  3. Refresh your workspace in Eclipse.
