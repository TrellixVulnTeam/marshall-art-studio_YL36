Setting Up Your Portal 5 Project

NOTE:

We expect that only one member of the team will go through the setup process and subsequent publish to SVN. Ideally, this should be performed at the start of a project. Team members will check out the project from subversion and import it into Eclipse as a Maven project. Refer to the following procedure for more information: Importing Maven Projects.

SDK core is distributed as a Maven archetype. You can refer to Introduction to Archetypes on the Maven website for more information; however, in simple terms, think of a Maven archetype as a Getting Started project. We publish this archetype on our Nexus repository. If you do not have access to the Nexus repository, refer to the Alternate Steps: Consuming the Archetype from an FTP Site topic for more information.

Perform the following to consume a published archetype from a command line:

(Refer to the Generate Project Using an Alternative Catalog on the Maven website for more information.)

  1. Run the following command:

    2. mvn -U archetype:generate -DarchetypeCatalog=http://10.21.39.108:8081/nexus/service/local/repositories/releases/content/archetype-catalog.xml

  2. Follow the prompts below to select the 'portal archetype' and configure your new project.
    Refer to the Guide to naming conventions on groupId, artifactId and version topic on the Maven website for more information on Maven's conventions for naming your new project artifacts.
    1. The first prompt lists the archetypes available in the archetype catalog. Select the archetype named 'com:Agencyport.sdk.portal-archetype'.
    2. Provide a group ID for your project. It's a good convention to use a name that matches your project's top-level Java package name. For example, 'com.stateauto'.
    3. Provide an artifact ID for your porject. This will be the name of your project. For example, 'portal'.
    4. Provide a version number or press Enter to use the default. Refer to the Maven Version Number Rules topic on the Mojo website for more information.
    5. Provide a package name or press Enter to use the default:
    6. Enter 'Y' to confirm. This creates a portal project that is based on our archetype in the current directory.
