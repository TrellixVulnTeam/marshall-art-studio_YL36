AgencyPortal Web Application Archive Requirements

The following are the requirements for the AgencyPortal web application archive:

  • Some of AgencyPortal’s jar file dependencies, which are typically distributed as part of the product, are not needed on WebSphere environments since they will collide with jars provided as part of the container.
    • The following jar files should not be included in any AgencyPortal WAR or EAR file that is about to be deployed to a WebSphere container:
      • saaj.jar
      • geronimo-statx-api_1.0_spec-1.0.jar
      • xerces.jar
      • activation.jar
  • Ensure your database_prefix application property is set appropriately. While some application containers will bind JNDI names in the ‘java:comp/env/’ namespace by default, WebShere supports empty prefixes.
  • The cryptography library provided as part of IBM’s JDK mandates the usage of an alternate keystore file within our application. The SDK’s apbase.jar is packaged with this alternate keystore file so all you have to do is point to it within application properties [e.g., framework.properties or acsi.properties): security.keystore_filename=(asResource)keystore/ibmacsi.key].
