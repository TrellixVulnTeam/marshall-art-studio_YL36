DB2 Issue

If you receive the following exception and are using a DB2 database, you may experience the DB2 issue described in this section.

com.agencyport.security.provider.RoleProvider getRoles Standard SQL Exception Info for exception at level 0 - SQL State: 'null'; SQL Error Number: '0'; SQL Error Text: 'DSRA9110E: ResultSet is closed.';

Setup for a DB2 database is different to other vendors, as described and outlined below.

The issue that causes the error is that, by default, the application server configures the resultSetHoldability custom property with a value of 2 (CLOSE_CURORS_AT_COMMIT). This property causes DB2 to close its resultSet/cursor at transaction boundaries. Despite DB2â€™s default resultSetHoldability value of 1 (HOLD_CURSORS_OVER_COMMIT), the application server retains its own default value of 2 to avoid breaking compatibility with previous versions of the application server.

You can change the default if the need arises (the value of 2 needs to be changed to 1 through the WebSphere application server console).
