Request Requirements (GETs and POSTs)

A logon ID is necessary to satisfy the SolrFilter that guards the Solr REST interface. The servlet attempts to extract a security profile from the incoming request and, if one is not found, it looks for a logon_id parameter.

  • If either of these is found, the logon ID is acquired and processing continues.
  • If either of these is not found, a JSON error message generates and is sent back with a response.

If the logon ID extraction is successful, execution is handed off to an instance of ISolrManager, which is responsible for token generation and making the select or update query to Solr.
