Solr Security Servlet

The Solr Security Servlet class acts as a proxy for calls to Solr for client-side AJAX calls. This allows those calls to go through the appropriate security measures, such as extracting, or constructing, a security profile and using AgencyPortal’s API for Solr interaction, and provide a controlled mechanism for making Solr calls when those calls don’t originate from Java code.

Since the calls go through the ISolrManager instance, there’s no need to worry about generating a security token.

For all client-side requests attempting to reach the Solr REST interface, target the URL configured for the Solr security servlet.
