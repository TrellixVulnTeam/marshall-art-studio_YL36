Token Validity

To determine whether the token is valid, the isSolrTokenValid() method in SolrFilter is called on each incoming request.

  • If the token provided is either null or the timestamp cannot successfully parse out of the token (after decryption), the SecurityException displays with the following ResourceBundle message: message.error_token_invalid.
  • If the timestamp is successfully parsed out, then a comparison is done to ensure that the token was sent within the last five seconds (based on the serverâ€™s clock). A failed check displays a Security Exception with the following ResourceBundle message: message.error_token_expired.

If all checks pass, the validation method returns true to the caller.
