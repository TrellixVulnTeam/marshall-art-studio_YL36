Agencyport Software

2011-2018 Developed and styled dynamic web apps. I was primarily a front-end developer for the flagship product Agencyportal, I developed versions of the app for pre-sales demos and paying customers. As Agengyport's one-man art department, I also designed and developed minisites, product graphics, illustrations & infographics, marketing material, and internal art projects. I helped develop Agencyportal for the cloud in Angular JS and Angular Materials, and worked on the Zurb Foundation style guide project for Duck Creek.

Milestones

Skills grew from solely editing HTML and CSS to the more technical aspects (database, server, Eclipse settings). Helped streamline our Bootstrap/LESS environment, knocked out killer themes for pre-sales and clients (Electric Self Service, Hanover, Lancer, Merchants, State Auto). Became an vocal authority on best practices markup for responsive design and web accessibility, while developing a greater comprehension of client and user needs.

AgencyPortal

Product Dev

Helped develop and execute a responsive front-end after the 15-year-old Java app was completely overhauled from the ground up. The cleaner UX and visual design is based on Bootstrap, styles preprocessed with LESS (best sounding option at the time). Tasks included styling classic functionality for the modern shell, fixing web accessibility issues as they came up, and encouraging a best practices culture.

Pre-Sales

Brand themeing a demo version of Agencyportal for prospective customers, introducing semi-functional new features. Some themes were just adding logo and colors, others were far more elaborate. Created a process where most of the client specific styles were made by changing LESS variables. Unique challenges included working with occasionally poorly documented client branding, stylizing for multiple lines of business, and instructing sales team on how to make visual design changes on the fly.

Client Services

Customizing the app for paying customers, creating look and feel as close as possible to the their websites. Styling functionality not addressed in the base app meant creating new markup in JSP files, tweaking Maven settings and SQL databases, speaking directly to clients about functionality, working on remote platforms like AWS, and occasionally instructing client IT developers how to make changes when we weren't allowed to touch the code directly.
